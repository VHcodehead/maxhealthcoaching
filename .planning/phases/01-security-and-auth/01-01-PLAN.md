  <name>Task 1: Create email service, rate limiter, and Zod schemas</name>
  <files>src/lib/email.ts, src/lib/rate-limit.ts, src/lib/validations.ts</files>
  <action>
1. Install Resend: `npm install resend`

2. Create `src/lib/email.ts`:
   - Initialize Resend client with `process.env.RESEND_API_KEY`
   - Create a `buildBrandedHtml(options: {heading, body, ctaText, ctaUrl})` helper that returns MaxHealth-branded HTML email:
     - MaxHealth green (#059669), dumbbell icon placeholder or text logo, clean white background
     - Tone: "Hey [name]," style greeting
     - Footer: "MaxHealth Coaching -- Your Personal Training Partner"
   - Export `sendPasswordResetEmail(to: string, name: string, resetUrl: string)`: uses branded template with "Reset Your Password" heading, reset link CTA, 1-hour expiry note
   - Export `sendVerificationEmail(to: string, name: string, verifyUrl: string)`: uses branded template with "Verify Your Email" heading, verify link CTA
   - Export generic `sendEmail(to: string, subject: string, html: string)` as escape hatch
   - From address: `"MaxHealth Coaching <noreply@maxhealthfitness.com>"` (or use env var RESEND_FROM_EMAIL)

3. Create `src/lib/rate-limit.ts`:
   - Export `RateLimitConfig` type: `{ windowMs: number, maxAttempts: number }`
   - Export `rateLimit(key: string, config: RateLimitConfig): { success: boolean, remaining: number, resetIn: number }`
   - Implementation: Module-level `Map<string, { count: number, resetTime: number }>` 
   - On each call: if key not in map or expired, create fresh entry. If under limit, increment and return success. If at limit, return failure with resetIn.
   - Lazy cleanup: on each call, if entry is expired, delete it. Additionally, every 100 calls run a sweep of all expired entries to prevent memory leak.
   - Export preset configs: `LOGIN_LIMIT = { windowMs: 15*60*1000, maxAttempts: 5 }`, `SIGNUP_LIMIT = { windowMs: 60*60*1000, maxAttempts: 3 }`, `RESET_LIMIT = { windowMs: 60*60*1000, maxAttempts: 3 }`

4. Update `src/lib/validations.ts` -- add at the end (do NOT modify existing schemas):
   - `forgotPasswordSchema = z.object({ email: z.string().email() })`
   - `resetPasswordSchema = z.object({ token: z.string().min(1), password: z.string().min(8, 'Password must be at least 8 characters'), confirmPassword: z.string() }).refine(data => data.password === data.confirmPassword, { message: "Passwords don't match", path: ["confirmPassword"] })`

1. Add to `prisma/schema.prisma` (after User model, before Profile model):

```prisma
model PasswordResetToken {
  id          String   @id @default(uuid())
  userId      String   @map("user_id")
  hashedToken String   @map("hashed_token")
  expiresAt   DateTime @map("expires_at")
  createdAt   DateTime @default(now()) @map("created_at")

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@map("password_reset_tokens")
}

model EmailVerificationToken {
  id          String   @id @default(uuid())
  userId      String   @map("user_id")
  hashedToken String   @map("hashed_token")
  expiresAt   DateTime @map("expires_at")
  createdAt   DateTime @default(now()) @map("created_at")

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@map("email_verification_tokens")
}
```
