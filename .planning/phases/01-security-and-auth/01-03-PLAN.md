1. Update `src/app/api/auth/signup/route.ts`:
   - Add rate limiting at the top of the POST handler: extract IP from `request.headers.get('x-forwarded-for')` or `request.ip` or fallback to 'unknown'. Call `rateLimit(\`signup:\${ip}\`, SIGNUP_LIMIT)`. If not success, return 429 `{ error: "Too many attempts, try again later" }`.
   - After the existing `prisma.$transaction` block that creates user + profile, add email verification token generation:
     - Generate token: `crypto.randomBytes(32).toString('hex')`
     - Hash: `crypto.createHash('sha256').update(token).digest('hex')`
     - Store: `prisma.emailVerificationToken.create({ data: { userId: user.id, hashedToken, expiresAt: new Date(Date.now() + 24*60*60*1000) } })` (24-hour expiry for verification)
     - Build URL: `\${process.env.NEXT_PUBLIC_APP_URL}/api/auth/verify-email?token=\${token}`
     - Call `sendVerificationEmail(parsed.data.email, parsed.data.full_name, verifyUrl)`
   - Keep existing response format. Email send failure should NOT block signup (wrap in try/catch, log error but still return success).

2. Create `src/app/api/auth/verify-email/route.ts`:
   - GET handler (user clicks link in email, so it must be GET)
   - Read `token` from URL search params
   - If no token: redirect to `/verify-email?status=error&message=Invalid+verification+link`
   - Hash the token: `crypto.createHash('sha256').update(token).digest('hex')`
   - Find EmailVerificationToken where hashedToken matches AND expiresAt > new Date(), include user
   - If not found/expired: redirect to `/verify-email?status=error&message=Expired+or+invalid+link`
   - If found: In transaction, set `user.emailVerified = true` and delete the token
   - Redirect to `/verify-email?status=success`

   - If status=error: show error card with AlertCircle icon, "Verification failed" heading, the message param or default "Something went wrong", and "Back to Dashboard" button
   - Match login page visual style (centered card, MaxHealth branding)
