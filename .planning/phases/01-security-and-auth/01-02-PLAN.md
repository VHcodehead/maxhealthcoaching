<task type="auto">
  <name>Task 1: Create forgot-password and reset-password API routes</name>
  <files>src/app/api/auth/forgot-password/route.ts, src/app/api/auth/reset-password/route.ts</files>
  <action>
1. Create `src/app/api/auth/forgot-password/route.ts`:
   - POST handler accepting `{ email }` validated with `forgotPasswordSchema`
   - Apply rate limiting: `rateLimit(\`reset:\${email}\`, RESET_LIMIT)` -- return 429 with "Too many attempts, try again later" if blocked
   - Look up user by email (include profile for name). If user not found, STILL return 200 with generic message (no email enumeration)
   - If user found:
     - Delete any existing PasswordResetToken records for this user (clean slate)
     - Generate token: `crypto.randomBytes(32).toString('hex')`
     - Hash token: `crypto.createHash('sha256').update(token).digest('hex')`
     - Store: `prisma.passwordResetToken.create({ data: { userId, hashedToken, expiresAt: new Date(Date.now() + 60*60*1000) } })`
     - Build reset URL: `\${process.env.NEXT_PUBLIC_APP_URL}/reset-password?token=\${token}` (raw token in URL, hashed in DB)
     - Call `sendPasswordResetEmail(user.email, profile.fullName || 'there', resetUrl)`
   - Always return `{ message: "If that email is registered, we've sent a password reset link." }` with status 200

2. Create `src/app/api/auth/reset-password/route.ts`:
   - POST handler accepting `{ token, password, confirmPassword }` validated with `resetPasswordSchema`
   - Hash the incoming token: `crypto.createHash('sha256').update(token).digest('hex')`
   - Find PasswordResetToken where `hashedToken` matches AND `expiresAt > new Date()`
   - If not found or expired: return 400 `{ error: "Invalid or expired reset link. Please request a new one." }`
   - If found:
     - Hash new password: `bcrypt.hash(password, 12)`
     - In a Prisma transaction: update user's passwordHash, delete the token record
     - Return 200 `{ message: "Password reset successful. You can now sign in." }`
  </action>
