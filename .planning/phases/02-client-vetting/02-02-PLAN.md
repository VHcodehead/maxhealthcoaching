1. **src/lib/email.ts** — Add two new email functions following the established pattern (use buildBrandedHtml + sendEmail):

   `sendApprovalEmail(to: string, name: string, loginUrl: string)`:
   - Heading: "You've Been Accepted!"
   - Body: "Great news — you've been accepted into MaxHealth Coaching! Click the button below to log in and get started on your fitness journey."
   - CTA: "Get Started" linking to loginUrl
   - Subject: "Welcome to MaxHealth Coaching!"

   `sendRejectionEmail(to: string, name: string)`:
   - Heading: "Application Update"
   - Body: "Unfortunately, we're not able to take on new clients at this time. We'll reach out if a spot opens up."
   - CTA: "Visit Website" linking to the app URL (use `process.env.NEXT_PUBLIC_APP_URL || 'https://maxhealthfitness.com'`)
   - Subject: "MaxHealth Coaching — Application Update"

2. **src/app/api/admin/applications/route.ts** — GET endpoint:
   - Auth check: require session with role 'coach' or 'admin' (same pattern as /api/admin/clients)
   - Query: `prisma.profile.findMany({ where: { subscriptionStatus: 'pending_approval' }, orderBy: { createdAt: 'desc' } })`
   - Return: `{ applications: [...] }` with all profile fields including application fields
   - Use `toSnakeCase` from `@/lib/serialize` for response formatting (same as admin/clients)

3. **src/app/api/admin/applications/[id]/route.ts** — PATCH endpoint:
   - Auth check: require session with role 'coach' or 'admin'
   - Accept JSON body: `{ action: 'approve' | 'reject' }`
   - Find profile by id param: `prisma.profile.findUnique({ where: { id } })`
   - If not found or subscriptionStatus !== 'pending_approval': return 404
   - **Approve**: Update `subscriptionStatus` to `'none'` (allows normal pricing/checkout flow). Send approval email (non-blocking try/catch, same pattern as signup verification email).
   - **Reject**: Update `subscriptionStatus` to `'rejected'`. Send rejection email (non-blocking try/catch).
   - Return: `{ success: true, action }` with 200
  </action>

    <automated>npx tsc --noEmit 2>&1 | head -20</automated>
  </verify>
  <done>GET /api/admin/applications returns pending applications. PATCH /api/admin/applications/[id] approves (sets 'none' + sends email) or rejects (sets 'rejected' + sends email). Both endpoints require coach/admin role. TypeScript compiles cleanly.</done>
</task>
