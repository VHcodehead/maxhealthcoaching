<task type="auto">
  <name>Task 2: Harden webhook handler with idempotency, logging, and edge case handling (PAY-02)</name>
  <files>src/app/api/webhooks/stripe/route.ts</files>
  <action>
Modify `src/app/api/webhooks/stripe/route.ts` to add production resilience:

1. **Add webhook event logging** at the top of the switch block (before individual case handling):
   ```typescript
   console.log(`[Stripe Webhook] ${event.type} | ${event.id} | customer: ${(event.data.object as any).customer || 'N/A'}`);
   ```
   Use `console.log` (not a DB table -- per Claude's discretion, console logging is sufficient for initial production debugging).

2. **checkout.session.completed** -- Handle missing user_id gracefully:
   - Current code silently skips if `!userId`. Add a `console.error` warning:
     ```typescript
     if (!userId) {
       console.error(`[Stripe Webhook] checkout.session.completed missing user_id in metadata | session: ${session.id} | customer: ${customerId}`);
       break;
     }
     ```
   - Add idempotency: Before updating, check if profile already has this subscriptionId:
     ```typescript
     const existing = await prisma.profile.findFirst({
       where: { userId },
       select: { stripeSubscriptionId: true },
     });
     if (existing?.stripeSubscriptionId === subscriptionId) {
       console.log(`[Stripe Webhook] Idempotent skip: checkout.session.completed for user ${userId}`);
       break;
     }
     ```

3. **customer.subscription.updated** -- Add idempotency guard:
   - Before updateMany, fetch current profile status:
     ```typescript
     const existing = await prisma.profile.findFirst({
       where: { stripeCustomerId: customerId },
       select: { subscriptionStatus: true },
     });
     if (existing?.subscriptionStatus === status) {
       console.log(`[Stripe Webhook] Idempotent skip: subscription.updated status already '${status}' for customer ${customerId}`);
       break;
     }
     ```
   - Fix the `current_period_end` type cast: Replace the `(subscription as unknown as { current_period_end: number }).current_period_end` hack with proper Stripe API access: `subscription.current_period_end` (this property exists directly on `Stripe.Subscription` -- the cast was unnecessary).

4. **customer.subscription.deleted** -- Add idempotency:
   - Check if already canceled before updating:
     ```typescript
     const existing = await prisma.profile.findFirst({
       where: { stripeCustomerId: customerId },
       select: { subscriptionStatus: true },
     });
     if (existing?.subscriptionStatus === 'canceled') {
       console.log(`[Stripe Webhook] Idempotent skip: subscription.deleted already canceled for customer ${customerId}`);
       break;
     }
     ```

5. **invoice.payment_failed** -- Add idempotency:
   - Same pattern: check if already `past_due` before updating.

6. **Add a default case** to the switch that logs unhandled event types:
   ```typescript
   default:
     console.log(`[Stripe Webhook] Unhandled event type: ${event.type}`);
   ```

7. **Wrap each case body** in a try/catch that logs the error but still returns 200 (Stripe retries on non-2xx, which could cause infinite retry loops for permanent errors like missing profiles):
   ```typescript
   case 'checkout.session.completed': {
     try {
       // ... handler logic
     } catch (err) {
       console.error(`[Stripe Webhook] Error handling checkout.session.completed:`, err);
     }
     break;
   }
   ```

Ensure the file compiles cleanly with `npx tsc --noEmit`.
