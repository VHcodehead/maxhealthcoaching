  <action>
Add the Message model to prisma/schema.prisma. Follow the existing conventions (uuid IDs, @map for snake_case, DateTime defaults, Cascade deletes).

```prisma
model Message {
  id         String   @id @default(uuid())
  senderId   String   @map("sender_id")
  receiverId String   @map("receiver_id")
  content    String
  read       Boolean  @default(false)
  createdAt  DateTime @default(now()) @map("created_at")

  sender   User @relation("sentMessages", fields: [senderId], references: [id], onDelete: Cascade)
  receiver User @relation("receivedMessages", fields: [receiverId], references: [id], onDelete: Cascade)

  @@map("messages")
}
```

Add the corresponding relation fields to the User model (alongside existing relations):
```prisma
sentMessages      Message[] @relation("sentMessages")
receivedMessages  Message[] @relation("receivedMessages")
```

After editing schema.prisma, run:
1. `npx prisma db push` to apply the schema change
2. `npx prisma generate` to regenerate the client

If the database is not available (no DATABASE_URL), just run `npx prisma generate` so TypeScript types are available.

Also add the Message type to `src/types/database.ts` if that file has manual type definitions. Check the file first — if it imports from Prisma client, the generate step handles it automatically.

  <action>
Create `src/app/api/messages/route.ts` with three handlers. All require authentication via `auth()` from `@/lib/auth`.

**GET /api/messages?user_id=xxx&count_only=true**
Two modes based on query params:

Mode 1 — Thread load (`user_id` provided):
- The authenticated user and `user_id` form a conversation pair
- Fetch messages where (senderId = me AND receiverId = user_id) OR (senderId = user_id AND receiverId = me)
- Order by createdAt asc
- Return `{ messages: Message[] }`

Mode 2 — Unread count (`count_only=true`, no user_id needed):
- Count messages where receiverId = me AND read = false
- If the caller is the coach (check profile.role === 'coach'), also return per-client breakdown: group unread messages by senderId, return `{ unreadTotal: number, unreadByUser: Record<string, number> }`
- If the caller is a client, return `{ unreadTotal: number }`

**POST /api/messages**
- Body: `{ receiverId: string, content: string }`
- Validate content is non-empty string, max 2000 chars
- Validate receiverId exists in users table
- Create message with senderId = authenticated user's id
- Return 201 with created message

**PATCH /api/messages**
- Body: `{ senderId: string }` — marks all messages FROM this sender TO the authenticated user as read
- `prisma.message.updateMany({ where: { senderId, receiverId: me, read: false }, data: { read: true } })`
- Return `{ updated: count }`

Authorization rules:
- Any authenticated user can use these endpoints
- Users can only read threads they are part of (enforced by the query filter)
- Users can only mark messages TO them as read (receiverId = me)
