**NOTIF-01: Plan ready email** — In `src/app/api/training-plan/route.ts` POST handler:
After the successful training plan save (this is the last step in plan generation), add a non-blocking email send. Fetch the user's profile to get their name and email:
```
// After successful plan save, before returning response
try {
  const profile = await prisma.profile.findUnique({ where: { userId: session.user.id }, select: { fullName: true, email: true } });
  if (profile?.email) {
    const appUrl = process.env.NEXT_PUBLIC_APP_URL || 'https://maxhealthfitness.com';
    sendPlanReadyEmail(profile.email, profile.fullName || 'there', `${appUrl}/dashboard`).catch(err => console.error('Plan ready email failed:', err));
  }
} catch (err) { console.error('Plan ready email lookup failed:', err); }
```
Import `sendPlanReadyEmail` from `@/lib/email`. Use `.catch()` on the promise (don't await) so the API response isn't delayed.

**NOTIF-02: Weekly check-in reminder cron** — Create new file `src/app/api/cron/checkin-reminders/route.ts`:
- Export GET handler
- Validate `CRON_SECRET`: compare `request.headers.get('authorization')` against `Bearer ${process.env.CRON_SECRET}`. Return 401 if missing/mismatched.
- Query all active clients: profiles with role='client', subscriptionStatus='active' (or any status that means they're approved and onboarded), onboardingCompleted=true. Join with user for email.
- For each client, check if they've already checked in this week using `hasCheckedInThisWeek` from `@/lib/checkin-schedule`. Get their latest check-in: `prisma.checkIn.findFirst({ where: { userId }, orderBy: { createdAt: 'desc' }, select: { createdAt: true } })`.
- For clients who haven't checked in, send `sendCheckinReminderEmail`. Use `Promise.allSettled` to send all emails concurrently.
- Return JSON with `{ sent: number, skipped: number, errors: number }`.
- App URL from `process.env.NEXT_PUBLIC_APP_URL`, checkin URL = `${appUrl}/checkin`.

**NOTIF-03: Coach check-in notification** — In `src/app/api/checkin/route.ts` POST handler:
After the successful check-in save (and after any macro adjustment logic), add a non-blocking email to the coach. Find the coach profile: `prisma.profile.findFirst({ where: { role: 'coach' }, select: { email: true, userId: true } })`. Also fetch the submitting client's profile for their name. Send `sendCheckinNotificationEmail` with the check-in data (weekNumber, weightKg, adherenceRating, notes) and review URL `${appUrl}/coach/clients/${session.user.id}`. Use `.catch()` pattern — don't await.

**NOTIF-04: Macro adjustment approved** — In `src/app/api/admin/macro-adjustments/[id]/route.ts` PUT handler:
After the successful approval transaction (where action === 'approve'), add a non-blocking email to the client. The adjustment record already has userId — fetch the client's profile for name and email. Send `sendMacroApprovedEmail` with `currentCalories` (old) and `approvedCalories` (new) from the adjustment. Use `.catch()` pattern.
